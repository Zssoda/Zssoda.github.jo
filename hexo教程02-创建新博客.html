Hexo init 初始化博客

文件夹下主要包含以下几个目录
|文件（夹）名|用途|
|—|:——:|
|_configure|配置文件,关于网站大部分配置参数都在这里修改，这也是以后用的最多的文件|
|scaffolds| 模板文件夹，内含3个md文件（post,draft,page），hexo new新建模板时，按照他来默认创建|
|themes|主题文件夹,hexo根据主题生成静态页面，要使用新主题是，也需要放在这个文件夹下|
|source|资源文件夹，存放用户资源，除posts文件外，开头命名为的文件将会忽略。文件夹中.md，和.html文件会被解析并放到publci,其他则直接拷贝过去|
|package.json|应用程序信息，可删除|

Hexo new 创建新博客

执行以下命令创建新博客

hexo new
  • 新建一篇文章。
  • 如果没有设置 layout 的话，默认使用 _configure.yml 中的 default_layout 参数代替。
  • 如果标题包含空格的话，需使用引号括起来。
title 标题

hexo默认以标题作为文件名册，但可以通过编辑_configure.yml 中的new_post_name参数来改变默认文件名称，例如：:year-:month-:day:title.md可以通过更方便用日期管理文章。
注意title是博客标题，文件名则指.md文件名称
|变量|描述|
|—|:–:|
|:title|标题（小写，空客会被换为短杠杆）|
|:year|年份，ex.2015|
|:month|月份,ex 04|
|:i_month|月份，ex 4|
|:day|日期，ex 07|
|:i_day|日期，ex 7|

layout 模板
scaffolds模板

结合上节，我们不难看出，这个[layout]实际就对应了scaffolds文件下的三个模板文件,其中pd.md如下图所示

其中这里提一下，draf这里草稿布局。这种布局在建立时会被保存到 source/_drafts 文件夹，可通过 publish 命令将草稿移动到 source/_posts 文件夹
hexo publish [layout] <title>

草稿默认不会显示在页面中，您可在执行时加上 --draft参数，或是把render_drafts参数设为 true 来预览草稿。

模板变量

其中Front-matter即是指文件最上方以 — 分隔的区域，用于指定个别文件的变量，因此，可以通过修改Front-matter,来对模板文件进行修改，可以预定义的变量如下
|参数|描述|
|–|:–:|
|layout|布局|
|title|标题|
|date|建立日期|
|updated|更新日期|
|comments|是否开启评论|
|categories|分类|
|tags|标签|
|permalink|覆盖文章网址|

自定义模板

因此我们也可以在scaffolds下自定义模板。

  • 执行这个命令，hexo就会尝试在scaffolds文件下寻找booknote.md，并根据其内容创建

  • 在自定义模板中我们能使用的变量仅有有layout,title,date这三Front_Matter

  • 自定义布局会创建到post文件夹下。

禁用模板

只需要把模板中Front-Matter 里的layout: 设为 false 即可
